# 15分钟实时价格采集定时任务设计文档

**日期**: 2026-03-17
**状态**: 已批准
**作者**: Claude + 用户协作

---

## 目标

将现有的实时价格采集定时任务从**每10分钟**改为**每15分钟**，继续使用现有的数据库表和查询接口。

---

## 背景

### 现有系统
- **文件**: `src/ecox/data/realtime.py`
- **当前配置**: 每10分钟在交易时段（9:30-11:30, 13:00-15:00）采集全市场A股实时数据
- **数据存储**: `a_share_real_time` 表
- **查询接口**: `MarketDataTool`

### 用户需求
> "实时价格查询应该做成一个定时任务, 每15分钟更新一次"

---

## 架构设计

### 修改范围
仅修改 `src/ecox/data/realtime.py` 的 CronTrigger 配置，其他部分保持不变。

### 技术方案

**文件**: `src/ecox/data/realtime.py`
**位置**: 第167-172行

**修改前**:
```python
trigger_normal = CronTrigger(
    second=0,
    minute='*/10',  # 每10分钟
    hour='9-11,13-14',
    timezone='Asia/Shanghai'
)
```

**修改后**:
```python
trigger_normal = CronTrigger(
    second=0,
    minute='*/15',  # 每15分钟
    hour='9-11,13-14',
    timezone='Asia/Shanghai'
)
```

### 数据流

```
APScheduler (15分钟间隔)
    ↓
akshare.stock_zh_a_spot() - 全市场A股实时数据
    ↓
insert_stock_data() - 数据清洗和转换
    ↓
DataCollectionService.save_realtime_data() - ORM批量保存
    ↓
a_share_real_time 表 - PostgreSQL存储
    ↓
MarketDataTool - Agent查询接口
```

## 触发时间表

### 修改后触发时间点

**上午时段 (9:00-11:30)**:
- 09:15 (预刷新)
- 09:30 (开盘)
- 09:45
- 10:00
- 10:15
- 10:30
- 10:45
- 11:00
- 11:15 (最后)

**下午时段 (13:00-15:00)**:
- 13:00 (开盘)
- 13:15
- 13:30
- 13:45
- 14:00
- 14:15
- 14:30
- 14:45 (最后)

**总计**: 每天17次触发（9次上午 + 8次下午）

### 时间窗口说明
- `hour='9-11,13-14'` 表示9-11点和13-14点
- `minute='*/15'` 表示每15分钟（0, 15, 30, 45分）
- **9:15触发**: 在开盘前15分钟进行预刷新，确保开盘时数据最新

### 错误处理
- 采集失败时记录错误日志到 `update_log` 表
- 自动跳过失败任务，等待下一个15分钟触发
- 不影响 Agent 查询历史数据

## 性能影响

### API调用频率
- **修改前**: 每天 ~24次（10分钟间隔）
- **修改后**: 每天 ~17次（15分钟间隔）
- **减少**: ~29%

### 数据库负载
- 每次触发写入约5000条记录
- 每天写入约 5000 × 17 = 85,000 条
- 使用批量插入优化性能

---

## 风险评估

### 低风险
- 仅修改时间间隔，不改变核心逻辑
- 保持现有数据表结构不变
- 保持现有查询接口不变

### 注意事项
- 确保非交易日不触发（依赖 `pandas_market_calendars`）
- 监控 akshare 接口限流（CALL_INTERVAL=360秒）
- 避免在14:45后触发（已通过 hour='9-11,13-14' 限制）
